ODBC vs JDBC in Python

Oct 02, 2019

Things on this page are fragmentary and immature notes/thoughts of the author. It is not meant to readers but rather for convenient reference of the author and future improvement.

Overall speaking, Python has better ODBC support than JDBC support. Most database related packages in Python support or rely on ODBC. Currently, ODBC packages also have richer feathers than JDBC packages. However, it is not an easy job to install and configure ODBC drivers for a non-open source databases.

ODBC vs JDBC

JDBC Packages in Python

  1. JayDeBeApi is currently the first choice even thought it is not fully DB-API compliant. It works on both Python 2 and Python 3.

  2. py4jdbc (based on Py4j) is another JDBC package which claims to be faster than JayDeBeApi (based on JPype) but it is relative young compared to JayDeBeApi.

  3. Just ignore the JayDeBeApi3 package (for Python3 only).

  4. PyAthenaJDBC is a JDBC package specifically for Amazon Athena.

ODBC Packages in Python

  1. pyodbc and SQLAlchemy are general purpose packages relying on ODBC.

  2. There are lots of database specific packages relying on ODBC. For example, teradata is a Python package for Teradata SQL which relies on ODBC or RESTful.

Database Modules

  1. json: JSON parsing.

  2. sqlite3

  3. jreese/aiosqlite

  4. JayDeBeApi: connect to databases using Java JDBC in Python.

  5. PyMySQL, MySQLdb

  6. PyMongo

  7. teradata

  8. pyodbc, pypyodbc: Python ODBC bridget.

  9. SQLAlchemy

  10. sqlalchemy-teradata
